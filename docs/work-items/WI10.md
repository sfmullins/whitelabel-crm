# WI10 — Platform API

WI10 introduces the versioned integration boundary on top of the WI8–WI9 identity, permission and audit model.

## Delivered scope

- authenticated `/api/v1` aliases for the supported organisation, contact, engagement, activity and reporting resources;
- loopback-trusted profile exclusion for all `/api/v1` and `/api/platform` requests;
- scoped API tokens with hash-only storage, expiry, revocation and disabled-owner enforcement;
- explicit platform permissions for API-token, webhook and diagnostic administration;
- authenticated OpenAPI 3.1 metadata;
- immutable versioned platform events;
- encrypted webhook signing secrets stored through the existing credential vault;
- durable webhook delivery rows, HMAC-SHA256 signing, bounded retry and dead-letter state;
- WI10 tests and smoke validation in the permanent `ci:verify` gate.

## Deliberate exclusions

- OAuth authorization-server behaviour;
- SAML or hosted identity-provider functionality;
- arbitrary webhook templates;
- arbitrary bulk table mutation;
- direct database access for integrations;
- arbitrary extension code;
- a second reporting or export engine.

The full WI10–WI12 programme plan is maintained in `docs/WI10-WI12-IMPLEMENTATION-PLAN.md`.
