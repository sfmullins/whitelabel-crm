# WI10 — Platform API

WI10 introduces the versioned integration boundary on top of the WI8–WI9 identity, permission, ownership, reporting and audit model.

## Delivered scope

- authenticated `/api/v1` access for the allow-listed organisation, contact, engagement, activity and reporting resources;
- a path/method contract guard preventing internal and legacy routes from becoming accidental public APIs;
- loopback-trusted profile exclusion for all `/api/v1` and `/api/platform` requests;
- scoped API tokens with one-time display, hash-only storage, expiry, transactional rotation, revocation and disabled-owner enforcement;
- explicit platform permissions for API-token, webhook and diagnostic administration;
- authenticated OpenAPI 3.1 metadata with request-ID and pagination contracts;
- ownership assignment and audit attribution for v1 mutations;
- immutable versioned platform events generated from successful CRM mutations;
- encrypted webhook signing secrets stored through the existing credential vault;
- webhook endpoint SSRF controls, including private/special-use blocking and DNS revalidation;
- durable webhook delivery rows, HMAC-SHA256 signing, bounded retry and dead-letter state;
- WI10 tests and smoke validation in the permanent `ci:verify` gate.

## Deliberate exclusions

- OAuth authorization-server behaviour;
- SAML or hosted identity-provider functionality;
- arbitrary webhook templates;
- generic bulk table/entity mutation;
- a second import, backup, reporting or export engine;
- direct database access for integrations;
- arbitrary extension code.
