# WI11 — Extension Platform

WI11 builds an upgrade-safe declarative extension layer on the merged WI10 API, event and security foundation.

## Implemented foundation

- strict versioned JSON manifest and package schemas;
- application-version compatibility checks;
- explicit capability approval;
- deterministic canonical manifest checksums;
- optional detached Ed25519 signature verification;
- extension, release, contribution, binding, migration and install-attempt registries;
- pre-migration backups for custom-field or custom-entity changes;
- transactional installation and upgrade;
- non-destructive enable/disable controls;
- namespaced custom fields and custom entities;
- safe custom-field/entity upgrade rules that prohibit identity or type mutation;
- system-managed bridging of pre-WI11 custom fields and objects;
- declarative storage for forms, views, navigation, themes, reports, workflow templates, event subscriptions, localisation and static-asset metadata;
- explicit extension read/manage permissions;
- immutable WI10 platform events for install, upgrade, enable and disable operations;
- separate extension export from any future data-purge operation;
- permanent `wi11:smoke` verification.

## Security constraints

Extensions cannot contain or execute JavaScript, SQL, shell commands or renderer bundles. Static assets are metadata-only in the current package API and must provide a bounded size and SHA-256 digest. Workflow contributions are templates restricted to the existing allow-listed workflow action model; package installation does not execute them.

Schema-affecting packages receive a verified SQLite backup before their transaction begins. A failed transaction leaves the prior active release intact and records the failed install attempt.

## Package outline

```json
{
  "manifest": {
    "formatVersion": 1,
    "packageKey": "vendor.case-management",
    "name": "Case Management",
    "version": "1.0.0",
    "application": { "minVersion": "1.0.0" },
    "capabilities": ["custom_fields", "themes"],
    "contributions": {
      "customFields": [],
      "customEntities": [],
      "forms": [],
      "views": [],
      "navigation": [],
      "themes": [],
      "reports": [],
      "workflowTemplates": [],
      "eventSubscriptions": [],
      "localisations": [],
      "assets": []
    }
  }
}
```

An optional `signature` object may contain an Ed25519 public key and detached signature over the canonical manifest JSON.

## Remaining WI11 work

- runtime frontend registries for forms, views, navigation, themes and localisation;
- report-definition execution through the existing reporting engine;
- explicit workflow-template instantiation rather than automatic activation;
- packaged static-asset ingestion and verified local storage;
- extension-owned data export and separately confirmed purge;
- install/upgrade UI and human-readable compatibility diagnostics;
- complete rollback/recovery tooling around backup restoration;
- final architecture/domain documentation and full WI4–WI11 review before merge.
