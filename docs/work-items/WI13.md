# WI13 — Enterprise packaging, distribution and release certification

WI13 is the final programme work item. It converts the WI12 application into a reproducible and supportable release system without weakening the published deployment-profile trust boundary.

## Non-negotiable boundaries

- Managed clients consume a signed WI12 deployment profile and never create an authoritative local CRM database.
- Standalone installations are independent and are labelled as such.
- SQLite server deployment is single-instance and single-writer; no clustered or active-active claim is made.
- Distributed artifacts contain no credentials, private signing material, live databases or backups.
- A release claim is made only where executable evidence exists; otherwise the limitation is published.
- All implementation and correction work is contained in PR #29 from `WI13-Enterprise-Release` to `main`.

## Stable gates

The user-facing gates are `wi13:structure`, `e2e`, `accessibility:verify`, `performance:verify`, `artifact:verify`, `container:verify`, `recovery:verify`, `release:integrity`, `release:verify`, `docs:verify` and `wi13:smoke`.
